EzBridge API Commands

Real-Time Events (Client-Sided Allowed)

Envie e receba pacotes normais de jogo livremente:

socket.emit("RemoteEvent", { type: "seu_tipo", data: "sua_payload" });

User Identification (Client-Sided Allowed)

Recupere o ID único permanente do emissor da requisição:

socket.emit("RemoteEvent", { type: "ebuserid_get" }); // O servidor responderá apenas para você com: // { type: "userid_response", userId: "ID_UNICO_DO_CLIENTE" }

EBstorage Commands (SERVER-SCRIPT AUTHORIZED ONLY)

APIs de banco de dados físico protegidas. Se houver um ServerScript ativo, o cliente comum não pode disparar esses comandos diretamente:

1. Adicionar/Atualizar Dado:

db.add("ebdata_add_pasta_subpasta_chave", data, userId);

2. Recuperar Dado:

db.fetch("ebdata_fetch_pasta_subpasta", userId);

3. Remover Dado:

db.remove("ebdata_remove_pasta_subpasta_chave", userId);

4. Criar Pasta de Organização:

db.add("ebdata_addfolder", { folderName: "sua_pasta" }, userId);

5. Limpar Tudo:

db.clear(userId);
